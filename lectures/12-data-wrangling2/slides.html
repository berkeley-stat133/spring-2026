Data Pipelines

library(tidyverse)

wizards <- data.frame(
  name = c("Harry", "Bellatrix", "Hermione", "Draco"),
  house = c("Gryffindor", "Slytherin", "Gryffindor",  "Slytherin"),
  height = c(1.78, 1.57, 1.65, 1.75),
  spells = c(60, 75, 70, 55)
)

Using base R data frame subsetting, create from wizards ‚Ä¶

  1. The average height of characters from Gryffindor.

Let‚Äôs look at three ways to solve this.

Nesting

summarize(filter(wizards, house == "Gryffindor"), mean(height))
  mean(height)
1        1.715


  • Must be read from the inside out üëé
  • Hard to keep track of arguments üëé
  • All in one line of code üëç
  • Only refer to one data frame üëç

Step-by-step

wizards2 <- filter(wizards, house == "Gryffindor")
summarize(wizards2, mean(height))
  mean(height)
1        1.715


  • Have to repeat data frame names üëé
  • Creates unnecessary objects üëé
  • Stores intermediate objects üëç
  • Can be read from top to bottom üëç

Data Pipelines in R


2016:

"magrittr" introduces %>%

2021:

R now provides a simple native forward pipe syntax |>. The simple form of the forward pipe inserts the left-had side as the first argument in the right-hand side call.

Using the Pipe Operator

wizards

Using the Pipe Operator

wizards |>
  filter(house == "Gryffindor")

Using the Pipe Operator

wizards |>
  filter(house == "Gryffindor") |> 
  summarize(mean(height))
  mean(height)
1        1.715


  • ü§∑
  • Can be read like an English paragraph üëç
  • Only type the data once üëç
  • No leftover objects üëç

group_by()

Flag the rows of a dataframe as belong to a group defined by a factor. For use in downstream operations.

group_by()

Flag the rows of a dataframe as belong to a group defined by a factor. For use in downstream operations.

Statefulness

What will this produce?

wizards |> 
  group_by(house) |> 
  mutate(height_z = (height - mean(height)) / sd(height)) |> 
  summarize(mean(height_z))
# A tibble: 2 √ó 2
  house      `mean(height_z)`
  <chr>                 <dbl>
1 Gryffindor         1.22e-15
2 Slytherin         -8.88e-16

Once grouped, a data frame stays grouped until reduced to one-row-per group or it is ungrouped.

Statefulness

What will this produce?

wizards |> 
  group_by(house) |> 
  mutate(height_z = (height - mean(height)) / sd(height)) |> 
  ungroup() |> 
  summarize(mean(height_z))
# A tibble: 1 √ó 1
  `mean(height_z)`
             <dbl>
1         1.11e-16

Shortcut: .by

Most "dplyr" functions have a .by argument to apply the function to the groups of another variable.

  • Not stateful
  • Good for single line operations
wizards |> 
    summarize(mean(height), .by = house)
       house mean(height)
1 Gryffindor        1.715
2  Slytherin        1.660
